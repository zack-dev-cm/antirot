# AntiRot v0.2.0 Reproducibility Log

This document captures the engineering work from the `v0.2.0` recovery pass so the same result can be reproduced without reading the original chat.

## Goal

Turn AntiRot from a placeholder-looking Markdown heuristic repo into a defensible public release with:

- materially better evidence checks
- passing local validation
- passing GitHub Actions CI
- a tagged GitHub release
- synced portfolio metadata

## Starting State

The main repo problems at the start of the pass were:

- the public README still read like a thin heuristic linter
- invalid citations could still inflate evidence coverage
- line-oriented parsing created avoidable Markdown false positives
- fenced code and front matter were not fully excluded from claim analysis
- numeric identifiers like `Section 3`, `GPT-4`, and `[32, 64]` could trigger claim checks
- the packaging surface still looked transitional
- there was no live repo-owned CI workflow on GitHub
- public collateral still showed old `v0.1.x` versioning

## Behavior Changes

The `v0.2.0` pass changed the linter in these ways:

- moved from line-by-line scanning to paragraph-aware Markdown parsing
- ignored fenced code blocks, front matter, blockquotes, tables, and reference-definition lines as claim prose
- supported inline links, raw URLs, DOIs, arXiv ids, footnotes, and in-document `References` sections as evidence anchors
- stopped counting unresolved citation markers as evidence when no references source was loaded
- added `citation-unverified` when a citation-like marker is present but cannot be resolved
- added `absolute-claim` for universal or zero-risk wording
- reduced false positives from numeric identifiers and numeric arrays

## Local Validation

These commands were used during the pass.

Run tests:

```bash
python3 -m pytest -q tests
```

Expected result on the shipped state:

```text
20 passed
```

Check syntax:

```bash
python3 -m py_compile antirot/*.py
```

Smoke the failing draft with references:

```bash
python3 -m antirot.cli lint examples/sloppy_paper.md \
  --references examples/references.md \
  --format text
```

Smoke the same draft without references to confirm `citation-unverified`:

```bash
python3 - <<'PY'
from antirot.linting import lint_markdown
print(lint_markdown("examples/sloppy_paper.md").to_text())
PY
```

Smoke the passing draft:

```bash
python3 -m antirot.cli lint examples/clean_paper.md \
  --references examples/references.md \
  --strict
```

Build from a clean virtualenv:

```bash
rm -rf build dist antirot.egg-info .tmp-build-venv
python3 -m venv .tmp-build-venv
source .tmp-build-venv/bin/activate
python -m pip install --upgrade pip build twine
python -m build
python -m twine check dist/*
pip install dist/*.whl
antirot lint examples/clean_paper.md --references examples/references.md --strict
```

## Release Sequence

The shipped release flow was:

1. Commit repo changes on `main`
2. Push repo changes to GitHub
3. Add the CI workflow on GitHub
4. Confirm the first Actions run passes
5. Create the release tag and GitHub release

Commands used locally:

```bash
git add .github/ISSUE_TEMPLATE/config.yml CHANGELOG.md CONTRIBUTING.md README.md \
  CITATION.cff docs/releases/v0.2.0.md docs/releases/v0.2.0-reproducibility.md \
  antirot/cli.py antirot/linting.py assets/antirot-social-card.svg \
  pyproject.toml tests/test_linting.py setup.py
git commit -m "Improve AntiRot evidence linting and packaging"
git push origin main
gh release create v0.2.0 --target main --title "AntiRot v0.2.0" --notes-file docs/releases/v0.2.0.md
```

## GitHub Workflow Scope Workaround

The local push path failed when the new workflow file was present because the active OAuth token did not have `workflow` scope.

Observed push failure:

```text
refusing to allow an OAuth App to create or update workflow `.github/workflows/ci.yml` without `workflow` scope
```

The workaround used during this pass was:

1. commit the non-workflow repo changes locally
2. push `main`
3. create [.github/workflows/ci.yml](/Users/zack/Documents/GitHub/github_stars_optimizer/.github/workflows/ci.yml) through the GitHub connector
4. fast-forward local `main` back to remote

Local resync command:

```bash
git pull --ff-only origin main
```
